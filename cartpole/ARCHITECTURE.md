# CartPole 强化学习训练框架 — 架构文档

## 项目概述

基于 Gymnasium 的 CartPole-v1 环境，实现三种强化学习算法的训练与对比。通过交互式菜单选择算法、调参、训练，并绘制奖励曲线。

## 目录结构

```

├── ARCHITECTURE.md         # 本文件
└── __pycache__/            # Python 缓存
```

## 模块详解

### ① Config（配置类）

所有超参数的集中管理。运行前修改即可调整算法和行为。

| 参数 | 类型 | 默认值 | 作用于 | 说明 |
|------|------|--------|--------|------|
| `algorithm` | str | `policy_gradient` | 全部 | 算法选择：`policy_gradient` / `sarsa` / `q_learning` |
| `n_episodes` | int | 500 | 全部 | 完整训练的总回合数 |
| `learning_rate` | float | 0.1 | SARSA/Q-Learning | 学习率 α |
| `pg_lr` | float | 0.1 | 策略梯度 | 策略梯度学习率 |
| `gamma` | float | 0.99 | 全部 | 折扣因子 γ |
| `epsilon` | float | 1.0 | SARSA/Q-Learning | 初始探索率（ε-贪婪） |
| `epsilon_min` | float | 0.01 | SARSA/Q-Learning | 最小探索率 |
| `epsilon_decay` | float | 0.995 | SARSA/Q-Learning | 每回合 ε 衰减系数 |
| `n_bins` | tuple | (6,6,12,8) | 全部 | 状态离散化格数（位置/速度/角度/角速度）|
| `use_baseline` | bool | True | 策略梯度 | 是否使用均值回报作为基线 |
| `hidden_size` | int | 128 | 策略梯度 | 神经网络隐藏层神经元数 |
| `seed` | int | 42 | 全部 | 随机种子 |

> **注意**：`n_bins` 和 `hidden_size` 在策略梯度算法中未使用（PG 使用连续状态输入），但保留作为统一接口。

### ② 环境与离散化工具

- **`make_env()`** — 每次调用创建一个全新的 CartPole-v1 环境实例
- **`_build_bins()`** — 根据 `n_bins` 为每个状态维度生成等距分箱边界（仅用于离散化方法）
- **`discretize()`** — 将 4 维连续状态向量映射为一个元组索引，如 `(3, 2, 7, 4)`

### ③ QLearningAgent（Q-Learning 离策略 TD 控制）

```
算法: Q(S,A) ← Q(S,A) + α [ R + γ·max_a Q(S',a) - Q(S,A) ]
```

- **Q 表**：形状 `(6,6,12,8,2)` 的全零张量
- **动作选择**：ε-贪婪策略（训练时以 ε 概率随机探索）
- **更新方式**：使用下一状态的最大 Q 值（离策略）
- **探索衰减**：每回合 `ε ← max(ε_min, ε × ε_decay)`

### ④ SarsaAgent（SARSA 在策略 TD 控制）

```
算法: Q(S,A) ← Q(S,A) + α [ R + γ Q(S',A') - Q(S,A) ]
```

- **与 Q-Learning 的区别**：使用实际执行的下一动作 A' 的 Q 值，而非最大 Q 值
- **训练循环**：需要先选动作 A，再在循环中选下一个动作 A'（两步动作选择）

### ⑤ PolicyGradientAgent（REINFORCE 策略梯度）

```
算法: θ ← θ + α G_t ∇_θ ln π_θ(A_t|S_t)
```

- **策略网络**：单隐层神经网络（tanh 激活），输入 4 维连续状态，输出 2 维 softmax 概率
- **动作选择**：按网络输出的概率分布采样（天然探索，无需 ε-贪婪）
- **更新时机**：每回合结束后，采集整条轨迹计算折扣回报，再做梯度上升

### ⑥ 训练循环

三个 `train_*()` 函数结构一致：
1. 创建智能体
2. 循环 n_episodes 次
3. 每回合：重置环境 → 循环选动作/执行/存储/更新 → 记录奖励
4. 返回 `rewards_per_episode` 列表

### ⑦ 绘图模块

- `plot_results()` — 画出原始奖励曲线 + 滑动平均 + 最终均值横线 + 参数文本框
- `moving_average()` — 滑动平均辅助函数
- 图表自动保存为 `training_curve_{算法}.png`

### ⑧ 交互入口

`interactive_main()` 提供：
1. 算法选择菜单（Q-Learning / SARSA / REINFORCE）
2. 参数查看与修改（输入编号修改、s 开始、m 返回）
3. 训练 → 自动绘图

### ⑨ 快捷入口

`run_demo(algorithm, n_episodes)` — 命令行直接调用，跳过交互菜单。

## 数据流（一次完整训练）

```
Config 设定参数
    │
    ▼
创建智能体（Q 表 or 网络 → 初始随机策略）
    │
    ▼
循环 n_episodes 次：
    ┌─────────────────────────────────────┐
    │  重置环境（CartPole-v1）              │
    │  do: 状态 → 选动作 → 执行 →          │
    │       奖励 + 下一状态 → 存储 → 更新    │
    │  while 终止                             │
    │  记录总奖励                          │
    └─────────────────────────────────────┘
    │
    ▼
返回 rewards 列表 → 绘制曲线 → 策略自动清空
```

## 如何更换算法

**方法一（交互式）**：
```
python cartpole_rl.py
→ 输入 1/2/3 选择算法
→ 输入编号修改参数
→ 输入 s 开始训练
```

**方法二（命令行）**：
```bash
python cartpole_rl.py q_learning 300     # Q-Learning, 300回合
python cartpole_rl.py sarsa 500          # SARSA, 500回合
python cartpole_rl.py policy_gradient 200  # REINFORCE, 200回合
```

## 如何调参

- **学习率 α**：太大不收敛，太小学得慢（常见范围 0.01~0.5）
- **折扣因子 γ**：0.95-0.99 适用于长期回报问题
- **探索率 ε**：初始 1.0，衰减到 0.01 左右。衰减过快可能过早收敛到次优策略
- **n_episodes**：CartPole 通常 200-500 回合可收敛（Q-Learning/SARSA 可能需要更多）
